请求表示 `nanhook` 入口收到的 HTTP 请求日志。
`nanhook` 评估每个请求是否应创建事件。如果请求符合事件条件，请求状态标记为“采纳”，并且创建事件；
否则请求被视为“驳回请求”。


您可以浏览有效时段内所有请求的历史记录，或筛选请求。此日志对于调试和排除请求故障以及它们生成的事件非常有用。

对于每个请求，请求`详情`会显示请求和响应数据：header, path, query, body。
